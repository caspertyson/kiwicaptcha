Use

Put this in your page somewhere:

<script src="https://kiwicaptcha.web.app/captcha.min.js"></script>

Put this in your site where you want the CAPTCHA:

<div class="captcha-box"></div>

Add this javascript, and change to your origin:

<script>
      window.addEventListener('message', function(event) {
          // Change "http://127.0.0.1:5500" to your site (no trailing slash)
          if (event.origin !== "http://127.0.0.1:5500") { 
              return; 
          }
          if (event.data.type === 'CAPTCHA_VERIFIED' && event.data.verified) {
              // e.g., enable/submit a form button etc. what happens after captcha is verified
          }
      }, false);
  </script>

Benefits:

Custom CAPTCHA fools a lot of bots

Googles reCaptcha owns 90+ percent of the market share in terms of CAPTCHA's.

Therefore, people looking to fool captcha forms and sign ins will benefit much more from creating automated bots to look for flaws in reCapture, or other more popular CAPTCHA's. This gives smaller CAPTCHA's like KiwiCAPTCHA better odds.

Kiwi-esque CAPTCHA increases customer engagement

How you do one thing is how you do everything!

If your customers see that you are paying attention to detail on such a level that you have tailored your websites CAPTCHA to your audience, they know that you will do that on whatever you're selling too!

Honey Potting

Trick the trickster.

The captcha.min.js file creates a honeypot input html element called "name". As this is hidden via CSS to the user, and not on the HTML, the bot will hopefully see this and add a name. If the API detects that the name input has been filled out, it must have been a bot, so it does not verify it as a human!

Timing and Unique Session ID

The following shows the actions done by KiwiCAPTCHA.

captcha.min.js > API: captcha.min.js sends a request to the API

API > captcha.min.js: API creates unique session ID, picks a question, stores the ID with the answer, and sends captcha.min.js the question

captcha.min.js > User: captcha.min.js receieves the question and displays it to the user.

User > captcha.min.js > API: the users answer, along with the unique session ID is sent back to the API

API > captcha.min.js: the API compares the users answer, along with the stored answer under the session, and returns a true or false. As well, if the session ID is older than 1 minute, the API will return false.

captcha.min.js: either the user is verified as human, or it will start the process from scratch!
